## 각 레이어가 하는 일

### models — DB 테이블 정의

파이썬 클래스로 DB 테이블을 표현한다.
`Base`를 상속받고 컬럼을 선언하면 SQLAlchemy가 실제 DB 테이블과 연결해준다.

```python
class StoreModel(Base):
    __tablename__ = "store"

    id: Mapped[int] = mapped_column(primary_key=True)
    region: Mapped[str] = mapped_column(String(50), index=True)
    year_quarter: Mapped[int] = mapped_column(Integer, index=True)
    trdar_div_code: Mapped[str] = mapped_column(String)
    trdar_div_name: Mapped[str] = mapped_column(String)
    trdar_code: Mapped[int] = mapped_column(Integer, index=True)
    trdar_name: Mapped[str] = mapped_column(String)
    service_code: Mapped[str] = mapped_column(String, index=True)
    service_name: Mapped[str] = mapped_column(String)
    store_count: Mapped[int] = mapped_column(Integer)
    # ...
```

9개 테이블 모두 `id`, `region`, `year_quarter`, `trdar_code`, `trdar_div_code`, `trdar_div_name`, `trdar_name` 이 7개 컬럼이 공통으로 들어간다.
지금은 그냥 각 파일에 똑같이 선언하고 있다.

### repositories — DB 쿼리

모델을 이용해서 실제 DB에 데이터를 넣거나 꺼낸다.

```python
class StoreRepository:

    async def bulk_insert(self, session: AsyncSession, rows: list[dict]) -> int:
        if not rows:
            return 0
        await session.execute(insert(StoreModel), rows)
        await session.commit()
        logger.info("[StoreRepository] bulk_insert %d rows", len(rows))
        return len(rows)

    async def count(self, session: AsyncSession) -> int:
        result = await session.execute(select(func.count()).select_from(StoreModel))
        return result.scalar() or 0

    async def find_by_trdar(self, session, trdar_code, year_quarter=None, service_code=None):
        stmt = select(StoreModel).where(StoreModel.trdar_code == trdar_code)
        if year_quarter:
            stmt = stmt.where(StoreModel.year_quarter == year_quarter)
        if service_code:
            stmt = stmt.where(StoreModel.service_code == service_code)
        result = await session.execute(stmt)
        return list(result.scalars().all())
```

`bulk_insert`와 `count`는 9개 레포지토리에 완전히 동일하게 들어가 있다.
`find_by_trdar`는 각 테이블마다 조건이 조금씩 다르다.

참고로 `ApartmentRepository`만 조금 다르다.
아파트 데이터가 많아서 한 번에 5,000개씩 나눠서 넣는 배치 처리를 했다:

```python
class ApartmentRepository:

    async def bulk_insert(self, session, rows, batch_size=5000):
        if not rows:
            return 0
        for i in range(0, len(rows), batch_size):
            await session.execute(insert(ApartmentModel), rows[i:i + batch_size])
            await session.commit()
        return len(rows)
```

다른 레포지토리랑 메서드 시그니처가 달라서 좀 찝찝하긴 한데 일단 동작한다.

### services — 비즈니스 로직

Reader로 CSV를 읽고, 컬럼명을 변환하고, Repository에 저장을 맡긴다.

```python
class StoreService:

    def __init__(self):
        self.repository = StoreRepository()

    async def ingest_store(self, session, year_quarter=None, region="seoul"):
        reader = MarketRadarReader(region)
        df = reader.load_store(year_quarter)
        df = df.rename(columns=KOREAN_TO_ENGLISH_COLUMN_MAP)
        df["region"] = region
        rows = df.to_dict(orient="records")
        count = await self.repository.bulk_insert(session, rows)
        logger.info(
            "[StoreService] ingest_store 완료 — region=%s year_quarter=%s rows=%d",
            region, year_quarter, count,
        )
        return count
```

이게 9개 서비스에 거의 똑같이 들어가 있다.
`load_store`, `KOREAN_TO_ENGLISH_COLUMN_MAP`, `StoreRepository` 이 세 군데만 다르고 나머지는 동일하다.

`ConsumptionService`와 `ApartmentService`는 float 컬럼에 NaN이 들어올 수 있어서
저장 전에 NaN을 None으로 바꾸는 처리가 추가로 있다:

```python
def _nan_to_none(rows):
    return [
        {k: (None if isinstance(v, float) and math.isnan(v) else v) for k, v in row.items()}
        for row in rows
    ]
```

이 함수가 `consumption_service.py`와 `apartment_service.py` 두 파일에 각각 정의되어 있다.

## 지금 구조의 솔직한 문제점

다 만들고 나서 코드를 보니까 눈에 밟히는 게 있다.

**1. 모델 파일마다 같은 7개 컬럼 반복**
`id`, `region`, `year_quarter`, `trdar_code` 같은 컬럼이 8개 파일에 똑같이 선언되어 있다.
만약 이 중 하나를 바꾸면 8개 파일을 다 열어야 한다.

**2. bulk_insert, count가 9개 파일에 복붙**
Repository마다 완전히 동일한 코드가 들어가 있다.
버그가 생기면 9개 파일을 다 찾아서 고쳐야 한다.

**3. ingest 흐름이 9개 서비스에 반복**
읽고 → 변환하고 → 저장하는 흐름이 전부 똑같은데 9번 작성됐다.

**4. _nan_to_none이 2개 파일에 각각 정의**
같은 함수인데 두 파일에 따로 있다.

**5. 메서드 이름이 다 다름**
`ingest_store`, `ingest_apartment`, `ingest_floating_population`...
컨트롤러가 각 서비스의 메서드 이름을 전부 외우고 있어야 한다.
